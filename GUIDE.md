- 화면에 나오는 검은 창의 **글자가 무서워 보여도 정상**입니다. 그건 n8n이 일하는 모습일 뿐입니다.

## 1. n8n과 워크플로우가 뭔가요?

- **n8n(엔에이트엔):** 여러 인터넷 서비스를 자동으로 이어주는 도구입니다.
- **워크플로우(workflow, 작업 흐름):** "이런 일이 생기면 → 저렇게 해라" 라는 **자동 규칙**입니다.
  - 예 1) "지메일에 새 메일이 오면 → 텔레그램으로 알림 보내기"
  - 예 2) "매일 아침 9시에 → 뉴스 사이트에서 제목을 모아 → 구글 시트에 저장"
- 즉, **반복되는 손작업을 컴퓨터가 대신** 하게 만드는 것입니다.
- 이 워크플로우는 **그림 블록을 선으로 잇는 방식**으로 만들기 때문에, 코딩을 몰라도 됩니다.

## 2. 사전 준비물 자세히

### 2-1. Windows 컴퓨터
- Windows 10 또는 11이면 됩니다. 특별한 사양은 필요 없습니다.
- 저장 공간은 **약 0.5~1GB** 정도 여유가 있으면 충분합니다.
- 이 `.bat` 도구는 **Windows 전용**입니다. (Mac·휴대폰에서는 동작하지 않습니다.)

### 2-2. 인터넷 연결
- **설치·업데이트**할 때 꼭 필요합니다. (인터넷에서 프로그램을 받아오기 때문)
- 일단 설치한 뒤 자동화를 만들 때는, 연결하려는 서비스에 따라 필요할 수 있습니다.

### 2-3. Node.js (노드)
- **Node.js**는 n8n이 동작하기 위한 **받침대 프로그램**입니다. (n8n이 Node.js 위에서 돌아갑니다.)
- **없어도 됩니다 — 이 키트가 자동으로 설치를 도와줍니다.**
  - 컴퓨터에 `winget`(윈도우 자동 설치 도구)이 있으면 **자동 설치**를 시도합니다.
  - 없으면 **공식 사이트(nodejs.org)** 를 열어 직접 설치하도록 친절히 안내합니다.
- 직접 설치하는 경우: 사이트에서 **"LTS"** 라고 적힌 큰 버튼을 누르고, 받은 파일을 두 번 눌러 **"다음(Next)"** 만 계속 누르면 됩니다.

> ⚠️ **Node.js를 새로 설치한 직후**에는, 열려 있던 검은 창을 **완전히 닫고** `.bat` 을 **다시 실행**해야 합니다. (그래야 Node.js가 인식됩니다.)

## 3. 다운로드 & 설치 (단계별)

### 3-1. 다운로드
1. 이 키트를 **압축파일(zip)** 로 받았다면: 마우스 오른쪽 클릭 → **"압축 풀기"**.
2. 풀린 폴더를 엽니다. 안에 `n8n_Manager_One-Click_Install-Run.bat` 가 보여야 합니다.
3. `사용설명서.txt` 도 **같은 폴더**에 함께 있는지 확인하세요.

> 💡 폴더를 **한글 경로**(예: 바탕화면)에 두어도 동작합니다. 문제가 생기면 `C:\n8n` 같은 짧은 영어 경로로 옮겨보세요.

### 3-2. 인터넷에서 받은 파일 "차단 해제" (필요할 때 한 번)
인터넷·메신저로 받은 파일은 Windows가 안전을 위해 잠가둘 때가 있습니다.
1. `.bat` 파일에서 **마우스 오른쪽 클릭 → "속성"**.
2. 맨 아래 **"차단 해제(Unblock)"** 체크박스가 보이면 **체크**합니다.
3. **"확인"**. (이 항목이 없으면 이미 풀려 있는 것이니 그냥 넘어가세요.)

### 3-3. 설치 (자동 추천)
1. `n8n_Manager_One-Click_Install-Run.bat` 를 **두 번** 누릅니다.
2. (처음 실행 시 **파란 경고창**이 뜨면 → 아래 3-4 참고)
3. 검은 창에 한국어 메뉴가 나오면, 키보드 **`0`** 을 한 번 누릅니다.
4. 화면 안내를 따릅니다.
   - Node.js가 이미 있으면 → 바로 n8n을 설치하고 **자동 실행**까지 합니다.
   - Node.js가 없으면 → 자동 설치를 시도하거나, 설치 사이트를 열어줍니다. (위 2-3 참고)
5. **"[완료] 설치가 끝났습니다!"** 가 보이면 성공입니다.

### 3-4. "Windows의 PC 보호" 파란 경고창이 뜨면 (처음 한 번)
- 인터넷에서 받은 `.bat` 을 처음 누르면 파란 큰 창(**"Windows의 PC 보호"**)이 뜰 수 있습니다.
- **바이러스라는 뜻이 아니라**, 처음 보는 파일이라 Windows가 한 번 확인하는 절차입니다.
- 처리법: **"추가 정보"** 글씨를 누른 뒤 → **"실행"** 버튼을 누릅니다. → 검은 창이 정상적으로 열립니다.
- 보통 **처음 한 번만** 필요합니다.

### 3-5. "관리자 권한"이 필요할 때
- 설치가 자꾸 실패하면, `.bat` 을 **마우스 오른쪽 클릭 → "관리자 권한으로 실행"** 으로 열어보세요.
- 백신(보안 프로그램)이 막거나 `.bat` 을 지우는 경우도 있습니다. 잠시 끄거나 "예외/허용"으로 등록한 뒤 다시 시도하세요.

## 4. 실행 방법 + 메뉴 0~9 전부 설명

검은 창에서는 **번호 키를 한 번** 누르면 바로 선택됩니다. (엔터를 안 눌러도 됩니다. 잘못된 키는 눌러지지 않으니 안전합니다.)

### `[0]` 처음이세요?
- 설치 + 실행을 **한 번에** 해줍니다. **처음 쓰는 분께 가장 추천**합니다.

### `[1]` 설치하기
- n8n만 설치합니다. (Node.js가 없으면 먼저 안내합니다.)

### `[2]` 업데이트하기
- 이미 설치된 n8n을 **최신 버전으로** 새로 고칩니다. 인터넷이 필요합니다.

### `[3]` 시작하기 (통로 5678)
- n8n을 **켭니다.** 가장 많이 쓰는 메뉴입니다.
- 켜지면 잠시 뒤 **브라우저가 자동으로 열립니다.** 안 열리면 주소창에 `http://localhost:5678` 입력.
- **"통로(포트)"** 란 n8n이 사용하는 **번호**입니다. 기본은 **5678**입니다.
- 5678이 이미 쓰이고 있으면, 화면에서 **① 그대로 5678 / ② 다른 번호로 켜기(추천) / ③ 메뉴로** 를 고를 수 있습니다.

### `[4]` 시작하기 (통로 번호 직접)
- 통로 번호를 직접 정해 켭니다. (예: `8080`)
- 잘 모르면 그냥 **Enter** — 기본 5678을 씁니다.
- 숫자가 아니거나 너무 큰 값(65535 초과)을 넣으면 자동으로 5678로 되돌립니다.

### `[5]` 백업 / 저장 폴더 열기
- `[1]` 지금 백업하기: 내 작업물을 **날짜별 복사본**으로 안전하게 저장합니다.
- `[2]` 저장 폴더 열기: 작업물이 저장된 폴더(`.n8n`)를 탐색기로 엽니다.
- `[3]` 뒤로 가기.

### `[6]` 문제 해결 (자동 점검)
- Node.js·npm·n8n·인터넷 상태를 한 번에 점검합니다. (인터넷은 `8.8.8.8` 에 신호를 보내 확인)
- **"이럴 땐 이렇게 하세요"** 처방을 보여줍니다.
- 결과를 **`n8n_진단결과.txt`** 파일로도 저장합니다. (도움 요청 시 첨부)

### `[7]` 삭제하기
- `[1]` 프로그램만 삭제: 만든 작업물은 **그대로 보관**. (추천)
- `[2]` 데이터까지 완전 삭제: 작업물도 **모두 지움**. **되돌릴 수 없습니다.** (두 번 확인 + 백업 권장)
- `[3]` 취소.

### `[8]` 끝내기
- 이 메뉴 창을 닫습니다. (n8n이 켜져 있다면 먼저 끄세요 — 6장 참고)

### `[9]` 사용설명서 보기
- 같은 폴더의 `사용설명서.txt` 를 메모장으로 엽니다.

## 5. 사용·작동 방법 (첫 화면·첫 자동화)

### 5-1. 첫 화면 — "계정 만들기"
1. `[3]` 으로 n8n을 켜면, 브라우저에 **"계정 만들기(Set up owner account)"** 화면이 나옵니다.
2. **이메일과 비밀번호를 직접 정해** 입력합니다.
   - 이 계정은 **내 컴퓨터 안에서만** 쓰는 것이라 아무 이메일이나 적어도 됩니다.
   - 단, **잊지 않게 메모**해 두세요. (10장 비밀번호 분실 대처 참고)
3. 만들고 나면 n8n 편집 화면이 나옵니다. 이제 자동화를 만들 수 있습니다.

### 5-2. "방화벽" 창이 뜨면
- **"Windows 보안 경고"** 창이 나오면 **"액세스 허용"** 을 누르세요. (내 컴퓨터 안 통신을 허용하는 것)

### 5-3. 첫 자동화(워크플로우) 만들기 — 아주 간단한 예
1. 편집 화면에서 **`+`** (새 워크플로우)를 누릅니다.
2. 시작 블록(트리거)을 하나 고릅니다. (예: 수동 실행 "Manual Trigger")
3. **`+`** 로 다음 블록을 잇습니다. (예: 간단한 알림/메모)
4. 위쪽의 **실행(Execute / Test)** 버튼으로 한 번 돌려봅니다.
5. 잘 되면 **저장(Save)** 을 누릅니다. → 저장한 자동화는 `.n8n` 폴더에 보관됩니다.

> 처음에는 "수동 실행 → 간단한 동작" 하나만 만들어 보며 익히는 걸 권합니다. 익숙해지면 메일·메신저·시트 연결로 넓혀가세요.

## 6. 끄기 / 다시 켜기

### 6-1. 끄기 (중요)
- n8n이 켜져 있는 동안 검은 창에는 글자가 계속 올라갑니다.
- 끄려면 **그 검은 창**에서 키보드 **`Ctrl` + `C`** 를 함께 누릅니다.
- **"종료하시겠습니까(Y/N)?"** 가 나오면 **`Y`** 를 누르고 Enter.

### 6-2. 검은 창을 그냥 닫아도 되나요?
- 창을 닫으면 n8n도 함께 꺼집니다. 큰 문제는 없지만, 위 `Ctrl + C` 방법이 더 깔끔합니다.

### 6-3. 다시 켜기
- `.bat` 을 다시 두 번 누르고 → `[3]` 을 누르면 됩니다.
- 참고: n8n은 **컴퓨터를 켤 때 자동으로 켜지지 않습니다.** 쓸 때마다 `[3]` 으로 직접 켜면 됩니다.

## 7. 백업 / 복원

### 7-1. 백업 (작업물 지키기)
1. 메뉴 `[5]` → `[1]` 지금 백업하기.
2. `C:\Users\(내이름)\n8n_backup_(날짜)` 폴더에 복사본이 생깁니다.
3. 그 폴더를 **USB나 클라우드**에 함께 옮겨두면 더 안전합니다.

### 7-2. 복원 (백업을 되돌리기)
- 작업물 원본 폴더는 `C:\Users\(내이름)\.n8n` 입니다.
- 복원하려면: n8n을 끈 상태에서, 백업 폴더의 내용을 `.n8n` 폴더에 덮어쓰면 됩니다.
- (잘 모르겠으면, 백업 폴더를 지우지 말고 그대로 두고 도움을 요청하세요.)

## 8. 삭제(제거) 방법

1. 메뉴 `[7]` 삭제하기.
2. **두 가지 중 선택:**
   - `[1]` **프로그램만 삭제** — 만든 작업물은 보관됩니다. (추천)
   - `[2]` **데이터까지 완전 삭제** — 작업물도 모두 사라지며 **되돌릴 수 없습니다.**
3. `[2]` 는 안전을 위해 **두 번 확인**합니다. 진행 전 반드시 `[5]` 백업을 권합니다.

> 참고: "프로그램만 삭제" 후 작업물 폴더(`.n8n`)는 그대로 남습니다. 나중에 다시 설치하면 이어서 쓸 수 있습니다.

## 9. 명령어 모음 (무엇을 하는지)

> 이 키트가 **내부에서 대신 실행**해 주는 명령들입니다. **직접 입력할 필요는 없습니다.** "무슨 일을 하는지" 참고용으로만 보세요.

| 명령어 | 하는 일 |
|---|---|
| `npm install -g n8n` | n8n을 설치합니다. |
| `npm update -g n8n` | n8n을 최신으로 업데이트합니다. |
| `npm uninstall -g n8n` | n8n을 삭제합니다. |
| `n8n start` | n8n을 기본 통로(5678)로 켭니다. |
| `n8n start --port=8080` | n8n을 정한 통로(예 8080)로 켭니다. |
| `n8n --version` | 설치된 n8n 버전을 확인합니다. |
| `node --version` | Node.js 버전을 확인합니다. |
| `npm --version` | npm(설치 도구) 버전을 확인합니다. |
| `winget install OpenJS.NodeJS.LTS` | (Node.js가 없을 때) Node.js를 자동 설치합니다. |

- **npm** 은 Node.js와 함께 설치되는 **프로그램 설치 도구**입니다.
- `-g` 는 "컴퓨터 전체에서 쓰도록 설치"를 뜻합니다.

## 10. 문제·오류 대처 (증상별)

> 가장 먼저 메뉴 `[6]` 문제 해결을 눌러 자동 점검하세요. 아래는 증상별 추가 해결법입니다.

### 🔵 처음 실행하니 "파란 경고창"이 떠요
- **"Windows의 PC 보호"** 창입니다. 바이러스 경고가 아니라 처음 보는 파일 확인 절차예요.
- **"추가 정보"** → **"실행"** 을 누르면 정상적으로 열립니다. (처음 한 번만)

### 🔴 백신이 `.bat` 을 지우거나 막아요
- 보안 프로그램이 처음 보는 `.bat` 을 의심하는 경우입니다.
- 잠시 백신을 끄거나, 이 파일을 **"예외/허용"** 목록에 넣은 뒤 다시 실행하세요.
- 받은 출처가 믿을 만하다면 안전합니다. (이 키트는 인터넷에서 n8n을 받아 설치만 합니다.)

### 🔴 "Node.js 가 없다 / 인식이 안 됨"
- 메뉴 `[1]`(또는 `[0]`)을 누르면 Node.js 안내가 나옵니다.
- 방금 설치했다면, 검은 창을 **완전히 닫고 다시 실행**하세요.

### 🔴 설치가 자꾸 실패해요
1. 인터넷 연결 확인.
2. `.bat` 을 **관리자 권한으로 실행**.
3. 백신/방화벽이 막는지 확인 (잠시 끄거나 예외 허용).
4. 회사/학교 네트워크는 차단이 있을 수 있어요. 가정 인터넷에서 시도해 보세요.

### 🔴 브라우저가 안 열려요
- 주소창에 직접 입력: `http://localhost:5678` (번호를 직접 정했다면 그 번호).
- 5~10초 기다린 뒤 **F5** 로 새로고침.

### 🔴 "통로(포트)가 이미 사용 중"
- 이미 n8n이 켜져 있을 수 있어요 → 주소창에 `http://localhost:5678` 입력.
- 다른 프로그램이 쓰는 거라면 → 메뉴 `[4]` 에서 다른 번호(예 8080)로 켜기.

### 🔴 "페이지를 열 수 없음" 이 떠요
- n8n이 아직 켜지는 중일 수 있어요. 5~10초 뒤 **F5**.
- 그래도 안 되면 검은 창에 빨간 글씨(오류)가 있는지 보고, 메뉴 `[6]` 결과 파일을 첨부해 도움 요청.

### 🔴 비밀번호를 잊었어요
- n8n 계정은 **내 컴퓨터 안**에만 있습니다.
- 해결: 작업물 폴더(`.n8n`)를 지우고 n8n을 다시 켜면 계정을 새로 만들 수 있습니다.
- ⚠️ 단, **만든 자동화도 함께 사라집니다.** 중요하면 먼저 `[5]` 백업하세요.

### 🔴 한글이 깨져 보여요 (검은 창)
- 이 `.bat` 은 한국어 Windows에 맞춰 저장돼 있습니다. **절대 메모장으로 열어 다시 저장하지 마세요** (한글이 깨집니다). 내용만 보려면 메뉴 `[9]` 또는 `사용설명서.txt` 를 이용하세요.

### 🟡 그래도 해결이 안 되면
- 메뉴 `[6]` → `n8n_진단결과.txt` 파일을 그대로 보내 도움을 요청하세요. (설치 실패는 대부분 인터넷·권한·백신 문제입니다.)

## 11. 자주 묻는 질문 (Q&A)

**Q. 검은 창의 영어/글자가 무서워요. 뭔가 잘못된 건가요?**
A. 아닙니다. n8n이 일하는 모습일 뿐이에요. 그대로 두세요.

**Q. 돈이 드나요?**
A. n8n을 **내 컴퓨터에서 직접 돌리는 것은 무료**입니다. (n8n 회사가 운영하는 유료 클라우드 서비스와는 다릅니다.)

**Q. 인터넷이 꼭 필요한가요?**
A. **설치와 업데이트**할 때 필요합니다. 켜서 쓰는 동안에는 연결하려는 서비스에 따라 필요할 수 있어요.

**Q. 내 정보(계정·작업물)가 외부로 전송되나요?**
A. 아니요. 이 도구와 n8n은 **내 컴퓨터 안에서만** 동작합니다. 작업물은 `.n8n` 폴더에만 저장됩니다.

**Q. 컴퓨터를 켜면 n8n도 자동으로 켜지나요?**
A. 아니요. **내가 `[3]` 으로 켤 때만** 동작합니다. (윈도우 시작 시 자동 실행 아님)

**Q. 다른 컴퓨터에서도 쓸 수 있나요?**
A. 네, 각 컴퓨터에 따로 설치하면 됩니다. 작업물을 옮기려면 `[5]` 백업 폴더를 새 컴퓨터의 `.n8n` 위치로 복사하세요.

**Q. 지우고 싶어요.**
A. 메뉴 `[7]` 을 누르세요. 작업물을 남기고 프로그램만 지울 수도, 전부 지울 수도 있습니다. (전부 지우기 전에 `[5]` 백업 추천)

**Q. 비밀번호를 잊었어요.**
A. 내 컴퓨터 안의 n8n 계정이라, 작업물 폴더(`.n8n`)를 지우고 다시 만들면 됩니다. 단, 만든 자동화도 함께 사라지니 중요하면 먼저 백업하세요.
